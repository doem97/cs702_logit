```
Hi! We're Zichen, Huihui, and Ni. Our project reproduces Log-it, published at CHI 2023 by Jiang, Sun, and Xia.

...

Console.log is the MOST common debugging tool for web developers, but honestly... the output is just a wall of plain text — hard to locate, hard to compare, and hard to understand. Log-it replaces it with interactive, color-coded Streams. Let me show you.

...

[excited] Alright, we run a nested loop tracking variables i, j, and their product. Two streams appear on the right — "Nested Loop" in blue and "Products" in red, each with a name, entry count, and a menu.

Now... switching to slider view, we step through entries one at a time. Changed values are highlighted in yellow — you can clearly see how i and j evolve across iterations.

...

Log-it can also attach a stream directly to a page element. Here, the stream floats right next to the target.

Resizing the element and logging again — the values update in place, keeping the log visually connected to what you're inspecting.

...

Now we generate fifteen logs across five categories. Instead of one messy output, each category gets its own color-coded stream.

Collapsing irrelevant streams and filtering for "warn" narrows it down instantly.

...

Next, we log a complex nested object three times.

Expanding a key in one entry expands it across ALL entries — that's synchronized interaction.

And double-clicking a key — like loginCount — highlights that property across EVERY entry, so we can compare a single field... without the clutter.

...

[thoughtful] So to recap... our prototype covers stream-based log organization, in-place sliding, element attachment, color-coded filtering, and property highlighting with synchronized interaction. All the data is live.

Next, we plan to add bar chart visualization, multi-property collection, LLM-powered log insight, and a timeline sparkline — plus a user study. Thanks for watching!
```
